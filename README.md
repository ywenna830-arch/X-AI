# 课迹

课迹是一个面向大学生的多源课程任务理解与自适应规划智能体。它把课程通知、文字、图片和文档中的学习任务整理成可确认、可追溯、可执行、可动态调整的个人学习计划。

## 解决的问题

大学课程任务常分散在课堂通知、课程平台、群消息、图片和文档中，学生容易遗漏截止时间、低估工作量或缺少执行节奏。课迹的第一版聚焦一个本地单用户闭环：录入任务 -> AI解析 -> 用户确认 -> 保存任务 -> 智能规划 -> 执行反馈 -> 动态重规划。

## 核心功能

- 连续对话式录入：普通消息由 DeepSeek 自然回复；课程通知、作业、考试、实验、论文、报告等内容会整理为待确认任务草稿。
- AI解析确认：结构化提取课程、任务、截止时间、材料、提交要求、预计时长和可信状态。
- 任务中心：保存、查看、编辑、删除、筛选和状态切换任务。
- 智能规划：设置每日可用时间，按截止时间、优先级和容量生成计划。
- 执行反馈：记录完成、部分完成、未完成，并重排剩余计划。
- 提醒与风险：展示今日、明日、即将截止、逾期、应用内提醒和容量不足风险。
- ICS导出：支持单任务和全部未完成任务导出日历文件。

## 技术架构

- 后端：Python Flask
- 前端：HTML、CSS、JavaScript
- 数据库：SQLite
- 配置：`.env`
- 示例配置：`.env.example`
- 第一版不使用 Vue、React 或复杂前端构建系统

## 安装步骤

建议使用 Python 3.10 或更高版本。阶段8验收已将 PyMuPDF 更新到支持当前 Python 3.13 环境的版本。

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
python -m pip install -r requirements.txt
copy .env.example .env
python run.py
```

如果 5000 端口被占用：

```powershell
$env:PORT=5050
python run.py
```

访问地址：

- 首页：http://127.0.0.1:5000/
- 添加任务：http://127.0.0.1:5000/tasks/new
- AI识别确认：http://127.0.0.1:5000/tasks/confirm
- 任务中心：http://127.0.0.1:5000/tasks
- 智能计划：http://127.0.0.1:5000/plan
- 健康检查：http://127.0.0.1:5000/health

## 环境变量

```text
FLASK_ENV=development
FLASK_DEBUG=0
PORT=5000
SECRET_KEY=replace-with-a-local-secret
DATABASE_PATH=
AI_API_KEY=
AI_API_BASE_URL=https://api.example.com/v1/chat/completions
AI_MODEL=replace-with-model-name
AI_TIMEOUT=20
AI_DEMO_MODE=0
MAX_IMPORT_FILE_BYTES=5242880
APP_TIMEZONE=Asia/Shanghai
```

API密钥只从环境变量读取，不能写入源代码。配置 `AI_API_KEY` 后，课迹助手会统一调用 DeepSeek 或兼容 OpenAI Chat Completions 格式的接口，并要求返回 `chat` 或 `task` JSON。未配置 `AI_API_KEY` 时，会启用页面明确标注的本地演示模式。

`DATABASE_PATH` 未配置时默认使用本地 `instance/tasks.sqlite3`；如需自定义SQLite文件位置，可设置该变量，应用会自动创建数据库父目录。

## 使用流程

1. 在首页或“课迹助手”页发送普通消息或粘贴课程通知。
2. 如通知来自图片、DOCX或PDF，先通过“+”附件入口提取文字。
3. DeepSeek 返回普通回复，或整理出任务预览卡片。
4. 点击“查看并确认”，进入AI识别确认页。
5. 用户检查并修改结构化字段。
6. 点击“确认并保存”，任务写入SQLite。
6. 在“智能计划”页设置可用时间并生成计划。
7. 保存计划后记录执行反馈，必要时一键重新规划。
8. 设置提醒或导出ICS日历文件。

## 录入说明

- 文字：直接粘贴通知，支持填写通知日期以解析“下周三”等相对时间。
- 图片：支持 PNG、JPG、JPEG；依赖 `pytesseract` 包和本机 Tesseract OCR 程序。
- Word：仅支持 DOCX，提取正文段落和表格文字。
- PDF：使用 PyMuPDF 提取文本型PDF；扫描型PDF会明确提示当前不做整份OCR。
- 上传文件只临时保存，解析成功或失败后都会清理。

## AI解析说明

真实AI接口需配置 `AI_API_KEY`、`AI_API_BASE_URL` 和 `AI_MODEL`。对话接口要求模型只返回合法JSON：普通交流为 `{"type":"chat","reply":"..."}`，任务信息为 `{"type":"task","reply":"...","task":{...}}`。任务字段仍会经过既有校验，解析结果不会直接保存，必须经用户确认。截止日期不明确时不擅自猜测，并标记为信息缺失。

## 智能规划说明

规划器基于任务截止时间、预计时长、优先级、状态和每日80%容量生成待确认计划。支持未来7天或14天、提前完成天数、周末加时和不可安排日期。已完成计划项会保留，不参与重排。

## 提醒和ICS导出

每个任务可设置提前7天、3天、1天、截止当天或自定义0-365天提醒。提醒和ICS导出统一使用 `APP_TIMEZONE`，默认 `Asia/Shanghai`。浏览器通知只能由用户主动启用，网页关闭后不保证可靠后台通知。

ICS支持：

- 任务详情页下载单任务ICS。
- 首页下载全部未完成任务ICS。
- 已完成任务和已完成计划项默认不进入导出。
- 无截止时间且无未完成计划项的任务会被跳过。

## 三组无隐私演示示例

课程论文：

```text
课程：学术写作
任务：课程论文终稿
请在2099-06-20 20:00前提交PDF到课程平台。需要材料：论文提纲、参考文献、课堂笔记。预计6小时。
```

编程作业：

```text
课程：软件工程
作业：任务管理模块优化
请在2099-06-18 18:00前提交代码仓库链接和测试说明。需要材料：源码文件、测试用例、运行截图。预计4小时。
```

在线课程：

```text
课程：在线学习专题
任务：完成第三单元视频、单元测验和学习笔记
请在2099-06-22 21:00前完成平台学习记录。建议材料：视频笔记、错题记录。预计150分钟。
```

## 2至3分钟演示流程

1. 15秒：说明课程任务分散、容易遗漏的问题。
2. 25秒：粘贴“课程论文”示例，展示对话式录入。
3. 25秒：点击解析，展示结构化确认页和可信状态。
4. 20秒：修改字段并保存任务。
5. 30秒：进入智能计划，设置每日可用时间并生成计划。
6. 20秒：展示容量预警或执行反馈。
7. 20秒：展示提醒、倒计时和逾期判断。
8. 15秒：下载单任务或全部未完成任务ICS。
9. 10秒：说明当前边界。

演示前准备：可选配置真实AI环境变量；若API不可用，使用本地演示解析模式和上述无隐私示例。

## 当前限制

- 当前是本地单用户应用，没有登录和云同步。
- 图片OCR依赖本机Tesseract程序，未安装时只能给出错误提示。
- 真实AI解析依赖外部API密钥和网络环境。
- 规划算法是规则式，不替代课程要求判断。
- ICS导入后的提醒效果取决于用户日历软件。

## 隐私与安全

- `.env`、`instance/`、数据库、上传临时文件、虚拟环境、缓存、日志和生成的ICS文件不应提交。
- 示例数据必须无真实姓名、学号、课程账号、邮箱、电话、学校内部信息和API密钥。
- 文件上传限制类型、大小和基础签名，并使用安全文件名。
- 测试优先使用临时数据库，不清空真实 `instance` 数据。

## 常见问题

- 依赖安装失败：确认 Python 版本，重新执行 `python -m pip install -r requirements.txt`。
- 图片OCR失败：安装本机 Tesseract OCR，并确认图片清晰。
- AI解析不可用：检查 `AI_API_KEY`、`AI_API_BASE_URL`、`AI_MODEL`，或使用本地演示解析模式。
- PDF没有文字：可能是扫描型PDF，当前阶段不做整份PDF OCR。
- ICS无法导入：确认任务有有效截止时间或未完成计划项，并使用支持标准ICS的日历软件。
